| 2 | 사업별 특성 미고려 | 비목 무관 일괄 체크 → **N/A 항목까지 부적정** (예: 공사-only 사업의 원천징수) |

---

## 2. 현행 SAFE (기준선)

- 검토: 유형 1개 + 파일 1건
- 체크리스트: 유형별 ①~④
- 판정: 유사도 + 고정 규칙검증
- 지식DB/Gemini: 검토 비교 단계 미사용
- 교차 검토: 없음

---

## 3. 목표 (To-Be)

- **검토 단위:** 사업(케이스) 1건 + 4종 자료 + 통합 체크리스트 1개
- **체크리스트:** 적용 조건·교차 대상·지식 근거 메타데이터
- **사업 프로필:** 비목·유형에 따른 N/A / 규칙 스킵

---

## 4. 권장 4단계

1. **요구·도메인·골든케이스** (코드 전 2~4주)
2. **사업 프로필 + N/A** (오판 감소, 3단계 전 필수)
3. **case 통합 검토 + 교차 항목**
4. **지식DB 연동 + 판정 고도화**

```
[1] 문서화 → [2] 프로필/N/A → [3] 통합 검토 → [4] 지식·교차 판정
```
