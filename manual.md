> **작성일**: 2026-06-14

## 1. 시스템 개요

### 1.1 목적 및 기능

SAFE는 지방보조금 관련 내부 자료(사업계획서·집행내역서·지출증빙·정산보고서)를 **로컬 PC에서만** 검토하여 부정수급 위험 징후를 탐지하는 시스템입니다.

| 기능 | 설명 |
|---|---|
| 체크리스트 관리 | 법령·지침 기반 점검항목 생성·관리 |
| 자료 검토 | 업로드 → OCR → 파싱 → 체크리스트 비교 → 판정 |
| 검토 이력 | 필터·페이징·CSV보내기 (감사 대응) |
| 보고서 출력 | HTML 보고서 자동 생성·인쇄 |

### 1.2 보안 원칙

```
🔒 SAFE 보안모드 — 내부자료 외부 전송 없음
```

- 모든 OCR·파싱·비교·저장은 **로컬 PC**에서만 수행
- 업로드 파일은 `uploads/` 폴더에만 저장 (URL 직접 노출 금지)
- Gemini API는 **공개자료(법령 PDF) 체크리스트 생성** 및 **설정 연결 테스트**에만 사용
- MariaDB는 `localhost:3306`만 연결

### 1.3 시스템 구성도

```
[브라우저] → FastAPI (localhost:8000)
                ├── OCR (Tesseract, pdfplumber) — 로컬
                ├── Parser — 로컬
                ├── Checker (FAISS + 임베딩) — 로컬
                ├── Logger → MariaDB (localhost)
                └── Reporter → reports/ — 로컬
```

## 3. 체크리스트 관리 (SCR-002)

### 3.1 체크리스트 조회

- 좌측 목록에서 자료유형별 체크리스트 선택
- 우측에 점검항목 상세 표시 (No·분류·점검항목·위험도·법령)

### 3.2 체크리스트 갱신 (법령 개정 시)

1. **[+ 새 생성]** 클릭
2. 자료유형 선택 (①~④)
3. 참조 공개자료 선택 (지방보조금법 PDF 등)
4. **[생성 시작]** — Gemini API 호출 (공개자료만)

> ⚠️ 내부 사업자료는 절대 Gemini에 전송되지 않습니다.

### 3.3 항목 수동 수정

- API를 통해 항목 수정 가능 (`PUT /checklist/api/{id}/item/{iid}`)
- 파일럿 기간 중 UI 인라인 수정은 추후 보완 예정

## 4. 자료 검토 방법 (SCR-003) ← 핵심

### 4.1 자료유형 선택

| 번호 | 유형 | 지원 형식 |
|---|---|---|
| ① | 사업계획서 | PDF, HWP |
| ② | 집행내역서 | PDF, Excel |
| ③ | 지출증빙자료 | PDF, 이미지 |
| ④ | 정산보고서 | PDF, Excel |

### 4.2 파일 업로드

- 드래그앤드롭 또는 클릭 업로드
- 최대 **100MB**
- 지원: PDF · HWP · Excel · JPG · PNG

### 4.3 검토 진행 대기

검토 시작 후 5단계 진행 표시:

1. 파일 수신 완료
2. OCR 처리
3. 항목 파싱
4. 체크리스트 비교
5. 결과 저장

- 2초 간격 자동 폴링
- 완료 시 결과 화면 자동 이동
- **예상 처리 시간**: 정산보고서 약 1~2분 (현재 PC 기준)

### 4.4 결과 확인 및 해석

결과 화면(SCR-004)에서 확인:

- **적합** ✅ — 기준 충족
- **주의** ⚠️ — 추가 확인 필요
- **부적합** ❌ — 즉시 조치 필요

---

## 5. 검토 결과 해석 (SCR-004)

### 5.1 판정 기준

| 판정 | 의미 | 조치 |
|---|---|---|
| 적합 (P) | 유사도 0.75 이상 또는 규칙 충족 | 없음 |
| 주의 (W) | 유사도 0.50~0.75 또는 경고 규칙 | 담당자 확인 |
| 부적합 (F) | 유사도 0.50 미만 또는 위반 규칙 | 즉시 조치 |

### 5.2 즉시 조치 필요 항목

빨간 강조 박스에 표시되는 항목:

- 사업기간 외 집행
- 반납계획 미기재
- 예산 외 집행
- 중복 증빙

### 5.3 보고서 출력

- **[보고서 출력]**: 브라우저 인쇄 (`Ctrl+P`)
- **HTML 다운로드**: `/review/{id}/report` 링크
- 저장 위치: `reports/` 폴더

## 6. 검토 이력 관리 (SCR-005)

### 6.1 이력 조회 및 필터링

- 자료유형 (①~④) 체크박스
- 기간·결과·사업명 키워드 검색
- 페이지네이션

### 6.2 CSV보내기 (감사 대응)

- **[CSV보내기]** 클릭
- 인코딩: `UTF-8 BOM` (Excel 한글 호환)
- 파일명: `SAFE_검토이력_YYYYMMDD.csv`

| 항목 | 평가 방법 |

- 수동 검토 대비 소요 시간 비교

## 부록 B. 문제 해결

| 증상 | 해결 |
|---|---|
| 서버 시작 실패 (포트 충돌) | `.env`에 `PORT=8001` 설정 |
