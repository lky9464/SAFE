# 04. 통합 체크리스트 분류 체계 (1차 확정)

> **상태:** 2026-07-01 1차 합의안  
> **전제:** `01_사업검토_정의서.md`, `03_사업프로필_정의.md`  
> **목적:** 유형별(①~④) 체크리스트를 **사업 1건 통합 검토**로 옮길 때 항목을 담을 **서랍(카테고리)** 정의

---

## 1. 분류 원칙

| 원칙 | 설명 |
|------|------|
| **업무 흐름 + 교차** | 예산→집행→증빙→정산 순서 + **4종 교차**를 별도 분류 |
| **비목 연계** | 항목마다 `required_budget_item`(편성목 코드) 태그 가능 |
| **자료 연계** | 항목마다 `required_docs`(①~④) 태그 필수 |
| **모듈 구조** | 하나의 거대 목록보다 **공통 코어 + 유형 모듈** 유지보수 |

---

## 2. 1차 분류 (7개)

| 코드 | 분류명 | 주요 점검 대상 | 필요 자료(대표) | 비고 |
|------|--------|----------------|-----------------|------|
| **C01** | 예산·사업계획 | 사업목적, 예산 편성, 비목 적정성, 자부담, 중복수혜 | ① | 계획서 중심 |
| **C02** | 집행·계약 | 사업기간, 용도 외 사용, 계약 적법성, 집행률 | ② | 집행내역 중심 |
| **C03** | 증빙·원천징수 | 증빙 완전성, 대조, 원천징수, 4대보험 | ③ (+② 교차) | ③ 없으면 N/A/W |
| **C04** | 정산·잔액·반납 | 실적·정산 제출, 잔액 반납, 회계감사 | ④ (+② 교차) | |
| **C05** | **교차대조** | 계획↔집행, 집행↔증빙, 정산↔집행, 사업기간 일치 | 2종 이상 | **3단계 핵심** |
| **C06** | 법령·지침 공통 | 보조금법·관리기준·조례 공통 의무 | ①~④ | 비목 무관 공통 |
| **C07** | 감사사례·특이 리스크 | 감사 지적 유형, 행사·시설 등 특수 | 상황별 | 지식DB 연결(4단계) |

## 3. 분류 ↔ 기존 SAFE 유형 매핑

| 기존 `data_type` | 주 배치 분류 | 교차 분류 |
|------------------|-------------|-----------|
| ① 사업계획서 | C01, C06 | C05(①+②) |
| ② 집행내역서 | C02, C06 | C05 |
| ③ 지출증빙 | C03 | C05(②+③) |
| ④ 정산보고서 | C04, C06 | C05(②+④, ①+④) |
| `checker.py` 규칙검증 | C01 또는 C02~C04 | 해당 비목·자료에 따라 |

## 4. 항목 메타데이터 (통합 체크리스트 1개 설계 시)

각 항목에 아래 필드를 부여한다.

| 필드 | 예시 | 용도 |
|------|------|------|
| `item_id` | CL-00123 | 고유 ID |
| `category` | C03 | 위 7분류 |
| `required_docs` | `["3"]` 또는 `["2","3"]` | 자료 미제출 시 N/A/W |
| `required_budget_item` | `["B101"]` | 편성 없으면 N/A |
| `check_method` | similarity / rule / cross / manual | 판정 방식 |
| `knowledge_ref` | 지식DB 파일·조항 | 4단계 근거 표시 |
